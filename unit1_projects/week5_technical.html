Ruby Classes

Phase 0 Unit 2: Week 5


Classes can be used to create new objects. Typically, you create a new class by:


# -> This symbol represents an indent.

class Name
-> some class describing the class behavior
end


You can also create things that the object knows and does. Things an object knows about itself are called instance variables. You'll want to initialize instance variable when a new object of class is created.


Here's an example:


class Person
-> def initialize(name, age)
-> -> @name = name
-> -> @age = age
-> end
end


Now let's talk about how to create things the object does. These are called methods. In the class Person above, let's create a hello method that returns the object's name and asks "What's your's?"


class Person
-> def initialize(name)
-> -> @name = name
-> end

-> def hello
-> -> puts "Hi, my name is #{@name}. What's your's?"
-> end
end


Now let's create a new Person object and call the hello method on it.


person_new = Person.new('Justin')
person_new.hello # => "Hi, my name is Justin. What's your's?"


You can create another new Person object by using the code above, replacing 'Justin' with another name and person_new with your own variable.


Now that we have a class, instance variables, and a method, I would like to raise an Argument Error: "You forgot to add a name." when someone forgets to add a name when creating a new Person object. This goes in the initialize area.


class Person
-> def initialize(name)
-> -> @name = name
-> -> raise ArgumentError.new("You forgot to add a name.")
-> end

-> def hello
-> -> puts "Hi, my name is #{@name}. What's your's?"
-> end
end


Therefore, if you create a new Person object without a name:


person_new = Person.new('') # => ArgumentError: "You forgot to add a name."
